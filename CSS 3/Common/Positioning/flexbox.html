Spaces 4 elements inside of the div, each element takes 25% of the space. The width is automatically set by the flex property, and if you try setting the width then it won't work because flex overrides it. You can set the height, though.

 
 
 
 

Now what if we have a 5th element but the flex is set to use 25% of the parent container? It's intelligent and knows to separate it out to only use 20% per element. What happens is that it tries to equal to 100%, so it adds the flex: 0.25 together as 0.25 + 0.25 + 0.25 + 0.25 + 0.25 = 1.25 and flex will set 1.25 as 100%.

 
 
 
 
 

There's only one element, but takes up only 25% of its parent container. Why? Didn't we just discuss that flex takes up the entire container by adding-up all of the flex? So why is it not taking up the entire container? This is because if using flex less than 1, then it intelligently doesn't take up the entire container and only takes the percent that was specified.

 

We can set the flex direction to be column so that the inner-divs can be stacked using columns. Remember, the height of each element is automatically set by the flex property when using the column direction. If you try setting the height property in CSS, then it won't work because flex overrides it. You can set the width, though.
